# Тестовое задание 

## О проекте
Проект выполнен в соответствии с требованиями
[технического задания](docs%2FTZ.docx)

## Структура проекта
```commandline
├───docs
│
├───output                  # пример выходных файлов
│ 
├───parsers
│       card_parser.py      # парсер деталей о товаре
│       catalog_parser.py   # парсер каталога товаров
│
├───utils
│       get_cookie.py       # получение cookies
│
└───xlsx_processing
        xlsx_formatter.py   # логика создания выходного xlsx
        xlsx_selection.py   # логика фильтрации данных

```

## Функционал
Скрипт реализует парсинг данных путём обращения к нескольким API:
 - API каталога товаров (CatalogParser) - получает страницы (по 100 элементов) 
с общими данными о товарах
 - API деталей о конкретном товаре (CardParser) - скрипт ищет и обращается
к API для получения детальной информации о товаре (card.json)
 - извлечение необходимой информации - происходит извлечение
из тела ответа обоих вышеуказанных API данных в соответствии с ТЗ
- формирование выходных xlsx-файлов

## Используемые компоненты
- selenium - для получения куки
- requests - для запроса очередной страницы каталога товаров
- aiohttp - для обращения к API с детальной информацией о товаре
- pandas - для выборки данных по условию

## Запуск
Для запуска парсера необходимо установить зависимости и 
запустить модуль main.py

1. ```commandline
   pip install -r requirements.txt
    ```
2. ```commandline
    python main.py
   ```

После запуска будут в терминале будут отображаться детальные логи
о процессе парсинга

Результатом парсинга являются 2 xlsx-файла:
 - parsed_data.xlsx - все данные по телу запроса
 - selection_parsed_data.xlsx - выборка данных по условию ТЗ

Среднее время работы парсера составляет 60 минут на 
4000 товаров при скорости интернета около 60 Мбит/с
